# Internationalization (i18n)

scanfence.com ships in 10 languages: English (default), Mandarin Chinese, Spanish, Arabic, Portuguese, Indonesian, French, Japanese, Russian, German.

## URL structure

- Default locale (English) lives at the site root: `/`, `/pricing/`, `/blog/`, etc.
- All other locales are prefixed: `/de/pricing/`, `/ar/blog/post-slug/`, `/zh/`, etc.
- The `x-default` hreflang points to the English root.

Locale codes: `en`, `zh`, `es`, `ar`, `pt`, `id`, `fr`, `ja`, `ru`, `de`.
For SEO we emit fuller hreflang codes where helpful: `zh-CN` for Chinese and `pt-BR` for Portuguese.

## Files

- `src/i18n/config.ts` ‚Äî locale list, RTL flag, native names, hreflang/og-locale codes.
- `src/i18n/t.ts` ‚Äî runtime `t(key, locale, vars?)` helper plus URL helpers
  (`localizedUrl`, `stripLocalePrefix`, `alternatesFor`, `resolveLocale`).
- `src/i18n/wp-cache.ts` ‚Äî eager-glob loader for translated WP pages and posts.
- `src/i18n/locales/{locale}.json` ‚Äî UI string catalogs. `en.json` is the
  source of truth; the others are AI-translated copies.
- `src/i18n/cache/wp-pages/{locale}/{slug}.json` ‚Äî translated WordPress page
  cache. Each file stores `{ sourceHash, translatedAt, title, content, ... }`.
- `src/i18n/cache/wp-posts/{locale}/{slug}.json` ‚Äî translated WordPress post
  cache (same shape, plus `excerpt`).
- `scripts/translate.ts` ‚Äî runs the AI translator. Idempotent: cached files
  whose `sourceHash` matches the current source are skipped.

## Adding or editing a UI string

1. Add the key + English value to `src/i18n/locales/en.json`.
2. Reference it in `.astro` files via `t('your.key', locale)`.
3. Run `npm run translate` to fan out to the 9 other locales.

The translator only re-translates keys that are missing or whose value still
equals the English source. To force a re-translation of every key:
`FORCE=1 npm run translate -- --force`.

## Re-translating WordPress content

```bash
npm run translate            # everything (UI + WP pages + WP posts)
npm run translate -- --ui    # UI strings only
npm run translate -- --wp    # WordPress content only
npm run translate -- --locale=de,fr   # restrict to specific locales
TRANSLATE_LIMIT=5 npm run translate -- --wp   # cap pages+posts per locale
```

If a WP post's source HTML changes upstream, its cached `sourceHash` no longer
matches and the next `npm run translate` re-translates that post in every
locale.

## API key

Set `ANTHROPIC_API_KEY` in a local `.env` file (gitignored). The script also
reads `WP_API_URL` (defaults to `https://cms.scanfence.com/wp-json/wp/v2`) and
`TRANSLATE_MODEL` (defaults to `claude-opus-4-7`).

## Adding a new locale

1. Append the code to `LOCALES` in both `src/i18n/config.ts` and
   `astro.config.mjs`.
2. Add a `LOCALE_META` entry and a `LOCALE_PROMPTS` entry (in `scripts/translate.ts`).
3. Add a fallback entry under `i18n.fallback` in `astro.config.mjs` (defaults to `en`).
4. Add a `<locale>.json` stub under `src/i18n/locales/`.
5. Import it in `src/i18n/t.ts` and wire it into `CATALOGS`.
6. `npm run translate` to populate the new locale.
7. `npm run build` to verify routes, sitemap, and hreflang clusters.

## URL slug policy

Slugs stay in English across all locales (e.g. `/de/blog/scanfence-launch/`,
not `/de/blog/scanfence-start/`). Reasons:

- Preserves inbound link equity.
- Keeps Yoast canonical rewrites in `src/lib/wordpress.ts` working without
  per-locale slug maps.
- Simplifies the slug ‚Üí cache lookup in `src/i18n/wp-cache.ts`.

## RTL

Arabic (`ar`) renders with `<html dir="rtl">`. CSS uses logical properties
(`margin-inline-start`, `border-inline-start`, etc.) so most layouts mirror
automatically. RTL-specific overrides live at the bottom of `src/styles/global.css`
under the comment `===== RTL mirroring =====`. Tabular data
(receipts, pricing comparison) is forced LTR with `direction: ltr` inside an
RTL page to keep timestamps and prices aligned.

## hreflang

`src/layouts/BaseLayout.astro` emits one `<link rel="alternate" hreflang>` per
locale for every page, plus an `x-default`. The `@astrojs/sitemap`
integration also emits hreflang clusters in `sitemap-*.xml` via the `i18n`
option in `astro.config.mjs`.
